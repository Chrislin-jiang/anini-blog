# Web 3D 交互设计方案

> 项目：lin-blog（Astro + React 个人主页）
> 设计目标：从现有浅色系 Toss 风格升级为深色霓虹赛博科技风，注入轻量 WebGL 3D 交互

---

## 一、设计概述

### 1.1 设计定位

将现有博客从「清新 Toss 风」升级为「高端数字创意 + 赛博科技风」，核心保持**内容可读性**与**交互克制感**，3D 作为氛围层存在，不喧宾夺主。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 渐进增强 | 3D 作为视觉增强层，无 WebGL 时优雅降级为 CSS 动画 |
| 性能优先 | 单页面模型面数控制在 5万面以内，首屏 3D 资源 < 500KB |
| 内容为王 | 文字、项目、博客内容始终清晰可读，3D 不遮挡主内容 |
| 交互克制 | 动效不连续自动播放，由用户滚动/鼠标触发，避免视觉疲劳 |

## 二、分区块 3D 交互设计

### 2.1 Hero 首屏 —— 「数字核心」

**现状**：居中头像 + 浮动几何装饰 + 渐变背景

**3D 升级方案**：

#### 视觉层：背景粒子星云
- **技术**：Canvas 2D / Three.js Points
- **效果**：
  - 800-1200 个浮空微粒子，颜色分布在青(#00F0FF)、紫(#B829F7)、粉(#FF0080) 三色
  - 粒子缓慢漂移（速度 0.2-0.5 px/frame），带呼吸式透明度变化
  - 粒子间距离 < 120px 时绘制连线，线条透明度随距离衰减
  - 鼠标移动时，周围 200px 半径内粒子产生轻微斥力散开，离开后缓慢回归原位
- **性能**：使用 `THREE.Points` 单 draw call 渲染，避免单独 mesh

#### 主体层：立体头像光环
- **技术**：CSS 3D + Framer Motion（保留现有头像）
- **效果**：
  - 头像外圈增加 3 层同心圆环，分别使用青/紫/粉色，带 `rotateX` / `rotateY` 倾斜
  - 圆环以不同速度缓慢旋转（8s / 12s / 20s），产生立体轨道感
  - 鼠标移动时，圆环产生视差倾斜（`transform: rotateX(y * 0.02deg) rotateY(x * 0.02deg)`）
  - 圆环使用 `border` + `box-shadow` 模拟霓虹发光，不引入额外 3D 库

#### 文字层：故障闪烁入场
- **技术**：纯 CSS / Framer Motion
- **效果**：标题「我是 Anini」入场时带轻微 glitch 闪烁（0.3s 内色彩通道错位 2-3 次），随后稳定为渐变霓虹色

#### 布局示意

### 2.2 About 关于我 —— 「知识星球」

**现状**：对话气泡列表 + 中间头像

**3D 升级方案**：

#### 核心：3D 技能星球
- **技术**：Three.js + `@react-three/fiber`
- **效果**：
  - 低多边形球体（Icosahedron，细分 2 级，约 1280 面），线框模式渲染
  - 球体表面分布 6-8 个发光节点，对应技能类别（React / TypeScript / Vue / 工程化 / 可视化 / AI）
  - 节点使用 Sprite 或 Html overlay 显示技术图标
  - 球体随滚动缓慢自转，鼠标拖拽可旋转视角（OrbitControls，禁用缩放）
  - 悬停节点时，该节点放大发光，其他节点变暗，底部显示技能详情文字
- **材质**：
  - 线框使用 `MeshBasicMaterial({ wireframe: true })`，颜色 `#00F0FF`，透明度 0.3
  - 节点使用 `MeshBasicMaterial` + 霓虹色，`emissive` 模拟自发光

#### 信息层：环绕文字轨道
- **效果**：
  - 球体赤道位置环绕一行细字「跨专业自学前端 · 滴滴前端工程师 · 掘金人气作者」
  - 文字沿圆形路径排列，随球体一起旋转
  - 使用 CSS 3D 环形文字或 Three.js `TextGeometry`（优先 CSS 方案减少开销）

### 2.3 TechStack 技术栈 —— 「能力矩阵」

**现状**：4 列技能卡片 + 圆环进度条

**3D 升级方案**：

#### 核心：六边形能力矩阵
- **技术**：CSS 3D Transform + Framer Motion
- **效果**：
  - 将现有圆环卡片改为**六边形蜂窝网格**排列，每个六边形为一个技能单元
  - 六边形使用 `clip-path: polygon(50% 0%, 100% 25%, 100% 75%, 50% 100%, 0% 75%, 0% 25%)`
  - 默认状态：六边形为深色半透明，边框发微弱霓虹光
  - 悬停状态：六边形沿 Z 轴抬起（`translateZ(20px)`），边框亮度增强，内部显示技能名称和掌握度百分比
  - 入场动画：六边形从中心向外依次翻转展开（`rotateY(90deg) → rotateY(0)`）

#### 装饰层：背景数据流
- **效果**：
  - 区块背景添加垂直下落的粒子雨（模拟数字代码雨），颜色为 `#00F0FF` 20% 透明度
  - 粒子为竖线形状（`1px × 10-20px`），随机长度和下落速度
  - 与主内容保持低对比，仅作为氛围装饰

### 2.4 Projects 项目展示 —— 「立体轮播」

**现状**：2 列玻璃卡片网格

**3D 升级方案**：

#### 核心：3D 卡片轮播（Carousel）
- **技术**：Framer Motion 3D + CSS `perspective`
- **效果**：
  - 项目卡片排列在 3D 弧形轨道上，当前卡片正对屏幕，两侧卡片向里旋转（`rotateY(±35deg)`）并缩小（`scale(0.85)`）
  - 支持左右拖拽 / 点击箭头切换，切换时使用 `spring` 物理动画
  - 卡片本身带 `preserve-3d`，正面为项目信息，悬停时轻微翻转露出背面标签云
  - 当前激活卡片底部投射霓虹色倒影（使用 `box-reflect` 或伪元素模拟）

#### 视觉层：项目封面悬浮
- **效果**：
  - 卡片封面图默认静止，悬停时产生「视差偏移」：图片在卡片内向鼠标反方向轻微移动（`transform: translate(x*0.05, y*0.05)`）
  - 封面边缘叠加渐变光晕，颜色取自项目主色

#### 数据映射
```
projects.map((project, i) => ({
  ...project,
  depth: Math.abs(i - activeIndex) * -100,    // Z轴深度
  rotateY: (i - activeIndex) * 35,            // Y轴旋转
  opacity: 1 - Math.abs(i - activeIndex) * 0.3 // 透明度衰减
}))
```

│    │   │ ← │ 项目 A  │ → │   │         │

### 2.5 Blog 博客列表 —— 「数据流」

**现状**：垂直文章列表，玻璃卡片样式

**3D 升级方案**：

#### 核心：倾斜列表 + 滚动联动
- **技术**：Framer Motion `useScroll` + `useTransform`
- **效果**：
  - 列表整体向屏幕内倾斜（`perspective: 1000px; rotateX(5deg)`），产生"数据面板"感
  - 滚动时，列表项根据视口位置产生 Y 轴位移和透明度变化（中间最亮，上下渐隐）
  - 每个列表项左侧增加「脉冲竖线」，颜色对应文章分类（技术/设计/随笔）

#### 交互：Hover 展开
- **效果**：
  - 悬停文章时，该项向 Z 轴抬起（`translateZ(30px)`），背景变亮，摘要文字展开
  - 左侧脉冲竖线变长并发出霓虹光晕

## 三、全局交互系统

### 3.1 平滑滚动

- **库**：`lenis`（轻量平滑滚动）
- **配置**：`lerp: 0.1`，支持 `scroll-behavior: smooth` 回退
- **联动**：滚动进度驱动 3D 场景相机位移、粒子速度变化

### 3.2 自定义光标

- **设计**：
  - 默认状态：空心圆环（`2px` 边框），霓虹青色，带微弱拖尾残影
  - 悬停可点击元素时：圆环缩小为实心圆点，颜色变为粉色
  - 进入 3D 交互区域时：光标外圈增加旋转圆环动画
- **实现**：CSS `cursor: none` + 绝对定位 `div` 跟随，使用 `requestAnimationFrame` + `lerp` 延迟跟随

### 3.3 页面转场

- **设计**：页面内锚点跳转时，不使用原生滚动，而是：
  - 当前区块内容向 Z 轴深处退去（`translateZ(-500px)` + `opacity: 0`）
  - 目标区块从 Z 轴前方进入（`translateZ(500px) → translateZ(0)`）
- **实现**：Framer Motion `AnimatePresence` + 自定义滚动劫持逻辑

### 3.4 滚动进度指示器

- **升级**：现有 `ScrollProgress` 从顶部细线改为：
  - 屏幕右侧边缘霓虹竖线，随滚动从下往上填充
  - 到达特定区块时，竖线对应位置闪烁对应颜色（青/紫/粉）

### 4.3 性能优化策略

| 策略 | 实现 |
|------|------|
| 懒加载 3D | Astro `client:visible` 指令，3D 组件进入视口才加载 |
| 降级方案 | `WebGL` 不可用时降级为 CSS 动画 + Canvas 2D |
| 离屏渲染 | 使用 `requestAnimationFrame` + 页面不可见时暂停渲染 |
| 模型优化 | 所有 3D 几何体使用低多边形，贴图压缩为 WebP |
| 着色器复用 | 统一粒子材质，使用 `ShaderMaterial` 减少 draw call |
| 防抖 resize | Canvas 尺寸变化使用 200ms debounce |

### 4.4 Astro 集成注意事项

1. **Three.js SSR 兼容**：所有 `@react-three/fiber` 组件必须添加 `client:only="react"` 或 `client:visible`，避免服务端渲染时报错
2. **分块打包**：配置 Vite `manualChunks`，将 `three` 相关库单独拆分为 chunk，避免首屏加载阻塞
3. **Tailwind 适配**：深色主题通过 CSS 变量注入，Tailwind `@theme` 中注册霓虹色值
