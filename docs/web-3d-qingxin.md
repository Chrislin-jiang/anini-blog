# Web 3D 林间清新交互设计方案

> 项目：lin-blog（Astro + React 个人主页）
> 设计目标：打造「林间清新 + 轻量 Canvas 2D」风格，营造田园自然、阳光明媚的数字花园体验
> 灵感来源：宫崎骏式田园风光、林间小路、蓝天白云、野花绿树

---

## 一、设计概述

### 1.1 设计定位

将现有博客打造为「林间清新数字花园」风格。视觉元素取材于真实自然场景——树叶、花瓣、萤火虫、蒲公英——以温暖、明亮、有机的方式融入页面。核心保持**内容可读性**与**情绪舒适感**，动效如林间微风般自然。

### 1.2 核心原则

| 原则     | 说明                                                          |
| -------- | ------------------------------------------------------------- |
| 自然模拟 | 粒子元素取材于自然形态（树叶、花瓣、萤火虫、蒲公英），避免机械几何 |
| 渐进增强 | 3D 作为视觉增强层，无 WebGL 时优雅降级为 CSS 柔和动画         |
| 性能优先 | 单页面模型面数控制在 3万面以内，首屏 3D 资源 < 400KB          |
| 内容为王 | 文字、项目、博客内容始终清晰可读，3D 不遮挡主内容             |
| 呼吸感   | 动效如自然风般轻柔缓慢，不急促、不刺眼，给人放松感            |

### 1.4 排版升级

- 英文字体：Inter → Inter + DM Sans（圆润友好）
- 中文字体：系统字体栈，标题字重 500-600（偏轻盈）

- 行高：整体行高提升至 1.7-1.8，增加呼吸感

## 二、分区块 3D 交互设计

### 2.1 Hero 首屏 —— 「林间微风」

**现状**：居中头像 + 浮动几何装饰 + 彩色气泡粒子

**升级方案**：

#### 视觉层：3D 有机变形球体 + 飘落粒子

- **技术**：Three.js + Canvas 2D 双层叠加
- **3D 层（Three.js）**：
  - **有机变形球体**：IcosahedronGeometry（细分 64 级），使用自定义 ShaderMaterial
    - 顶点着色器：3 层 Simplex Noise 叠加驱动顶点位移，产生有机流动变形效果
    - 片元着色器：Fresnel 边缘发光 + 基于位置的三色渐变（草木绿 → 晴空蓝 → 暖阳黄）+ 微光闪烁
    - 鼠标交互：鼠标位置影响顶点位移方向，球体随鼠标方向产生实时变形
    - 自动缓慢旋转，呼吸般的有机运动
  - **线框叠加层**：同步变形的半透明线框，增加科技感层次
  - **环绕粒子系统**（600个）：THREE.Points + 自定义着色器
    - 分布在球体外围球壳上，缓慢环绕旋转
    - 呼吸式大小脉动 + 透明度闪烁
    - 颜色：草木绿 / 嫩叶绿 / 晴空蓝 / 暖阳黄 / 白色
    - Additive Blending 产生柔和发光效果
- **2D 叠加层（Canvas 2D）**：
  - **飘落树叶**（30个）：贝塞尔曲线叶片形状，6 种绿色，带叶脉
  - **花瓣**（25个）：椭圆形花瓣，暖黄白色系
  - **蒲公英种子**（15个）：放射状细线 + 顶端小圆点，缓慢上浮
  - 全局微风系统影响所有 2D 粒子漂移
- **性能**：Three.js 单 draw call 粒子 + 低面数球体，Canvas 2D 约 70 个轻量粒子

#### 主体层：头像花环

- **技术**：CSS 3D + Framer Motion（保留现有头像）
- **效果**：
  - 头像外圈增加 2-3 层花瓣/叶片形态的装饰环，使用草木绿/暖阳黄色
  - 圆环以不同速度缓慢旋转（12s / 18s / 28s），如微风中的花环轻轻摇曳
  - 鼠标移动时，圆环产生视差倾斜（`transform: rotateX(y * 0.015deg) rotateY(x * 0.015deg)`），幅度更小更柔和
  - 圆环使用 `border` + `box-shadow` 模拟柔和光晕（非霓虹），无额外 3D 库

#### 文字层：渐显入场

- **技术**：纯 CSS / Framer Motion
- **效果**：标题「我是 Anini」入场时文字从下方轻柔上浮（`translateY(20px) → 0`，opacity 淡入），持续 0.8s，使用 `ease-out` 缓动
- 标题颜色为自然渐变（草木绿 #6DB33F → 晴空蓝 #5BA3D9），无 glitch 效果

### 2.2 About 关于我 —— 「知识花园」

**现状**：对话气泡列表 + 中间头像 + 深色线框星球

**3D 升级方案**：

#### 核心：3D 技能花球

- **技术**：Three.js + `@react-three/fiber`
- **效果**：
  - 低多边形球体（Icosahedron，细分 2 级，约 1280 面），**表面填充**模式（非线框），使用淡薄荷绿半透明材质
  - 球体表面分布 6-8 个「花苞节点」，对应技能类别（React / TypeScript / Vue / 工程化 / 可视化 / AI）
  - 节点使用 Sprite 或 Html overlay 显示技术图标，图标外有花瓣状装饰
  - 球体随滚动缓慢自转，鼠标拖拽可旋转视角（OrbitControls，禁用缩放）
  - 悬停节点时，该节点如花朵绽放般放大，其他节点变柔和（透明度降低），底部显示技能详情文字
- **材质**：
  - 球体使用 `MeshPhysicalMaterial`，颜色 `#8CC63F`，透明度 0.15，`transmission` 模拟玻璃质感，`roughness: 0.2`
  - 节点使用 `MeshBasicMaterial` + 自然色，`emissive` 模拟柔和自发光（非霓虹）

#### 信息层：环绕藤蔓文字

- **效果**：
  - 球体赤道位置环绕一行细字「跨专业自学前端 · 滴滴前端工程师 · 掘金人气作者」
  - 文字沿柔和波浪路径排列，随球体一起缓慢旋转
  - 使用 CSS 3D 环形文字或 Three.js `TextGeometry`（优先 CSS 方案减少开销）

### 2.3 TechStack 技术栈 —— 「能力林地」

**现状**：4 列技能卡片 + 圆环进度条 + 深色六边形矩阵

**3D 升级方案**：

#### 核心：圆形能力林地

- **技术**：CSS 3D Transform + Framer Motion
- **效果**：
  - 将现有卡片改为**圆形/圆角矩形**排列，如林间散落的石子，每个卡片为一个技能单元
  - 卡片使用大圆角（`border-radius: 24px`），背景为半透明白/淡绿渐变
  - 默认状态：卡片为浅色半透明，边框发微弱柔和光（天空蓝/薄荷绿）
  - 悬停状态：卡片如被微风吹动般沿 Z 轴轻轻抬起（`translateZ(15px)`），边框亮度柔和增强，内部显示技能名称和掌握度百分比
  - 入场动画：卡片从中心向外依次轻柔展开（`scale(0.8) → scale(1)` + opacity 淡入），带有错落的延迟

#### 装饰层：背景落叶/花瓣

- **效果**：
  - 区块背景添加缓慢飘落的叶片/花瓣粒子，颜色为 `#6DB33F` 20% 透明度 和 `#F4D03F` 15% 透明度
  - 粒子为不规则椭圆形状，随机大小和飘落速度，带轻微旋转
  - 与主内容保持低对比，仅作为氛围装饰

### 2.4 Projects 项目展示 —— 「立体画廊」

**现状**：2 列玻璃卡片网格 + 深色 3D 轮播

**3D 升级方案**：

#### 核心：3D 卡片画廊（Carousel）

- **技术**：Framer Motion 3D + CSS `perspective`
- **效果**：
  - 项目卡片排列在 3D 柔和弧形轨道上，当前卡片正对屏幕，两侧卡片向里轻微旋转（`rotateY(±20deg)`）并缩小（`scale(0.9)`）
  - 支持左右拖拽 / 点击箭头切换，切换时使用 `spring` 物理动画（阻尼更大，更柔和）
  - 卡片本身带 `preserve-3d`，正面为项目信息，悬停时轻微翻转露出背面标签云
  - 当前激活卡片底部投射淡色柔和倒影（使用伪元素 + 渐变模拟，非霓虹）

#### 视觉层：项目封面悬浮

- **效果**：
  - 卡片封面图默认静止，悬停时产生「视差偏移」：图片在卡片内向鼠标反方向轻微移动（`transform: translate(x*0.03, y*0.03)`），幅度更小
  - 封面边缘叠加柔和渐变光晕，颜色取自项目主色（自然色系）

#### 数据映射

```
projects.map((project, i) => ({
  ...project,
  depth: Math.abs(i - activeIndex) * -60,     // Z轴深度更浅
  rotateY: (i - activeIndex) * 20,             // Y轴旋转更小
  opacity: 1 - Math.abs(i - activeIndex) * 0.2 // 透明度衰减更缓
}))
```

### 2.5 Blog 博客列表 —— 「溪流」

**现状**：垂直文章列表，玻璃卡片样式

**3D 升级方案**：

#### 核心：倾斜列表 + 滚动联动

- **技术**：Framer Motion `useScroll` + `useTransform`
- **效果**：
  - 列表整体向屏幕内轻微倾斜（`perspective: 1200px; rotateX(3deg)`），产生"溪流流淌"感
  - 滚动时，列表项根据视口位置产生 Y 轴位移和透明度变化（中间最亮，上下渐隐）
  - 每个列表项左侧增加「水滴/叶片」装饰，颜色对应文章分类（技术/设计/随笔）

#### 交互：Hover 展开

- **效果**：
  - 悬停文章时，该项向 Z 轴轻轻抬起（`translateZ(20px)`），背景变亮，摘要文字展开
  - 左侧水滴装饰变大并发出柔和光晕（非霓虹）

## 三、全局交互系统

### 3.1 平滑滚动

- **库**：`lenis`（轻量平滑滚动）
- **配置**：`lerp: 0.08`（更柔和的阻尼），支持 `scroll-behavior: smooth` 回退
- **联动**：滚动进度驱动 3D 场景相机位移、气泡上升速度变化

### 3.2 自定义光标

- **设计**：
  - 默认状态：空心小圆环（`2px` 边框），天空蓝色，无拖尾残影（清新风格不追求科技感）
  - 悬停可点击元素时：圆环缩小为实心小圆点，颜色变为薄荷绿
  - 进入 3D 交互区域时：光标外圈增加轻柔的涟漪扩散动画（如水滴滴落）
- **实现**：CSS `cursor: none` + 绝对定位 `div` 跟随，使用 `requestAnimationFrame` + `lerp` 延迟跟随

### 3.3 页面转场

- **设计**：页面内锚点跳转时，不使用原生滚动，而是：
  - 当前区块内容向 Z 轴深处柔和退去（`translateZ(-300px)` + `opacity: 0`）
  - 目标区块从 Z 轴前方柔和进入（`translateZ(300px) → translateZ(0)`）
- **实现**：Framer Motion `AnimatePresence` + 自定义滚动劫持逻辑

### 3.4 滚动进度指示器

- **升级**：现有 `ScrollProgress` 从顶部细线改为：
  - 屏幕右侧边缘柔和渐变竖线（从天空蓝到薄荷绿），随滚动从下往上填充
  - 到达特定区块时，竖线对应位置变为对应自然色（绿/蓝/粉/黄）

| 策略        | 实现                                                  |
| ----------- | ----------------------------------------------------- |
| 懒加载 3D   | Astro `client:visible` 指令，3D 组件进入视口才加载  |
| 降级方案    | `WebGL` 不可用时降级为 CSS 动画 + Canvas 2D         |
| 离屏渲染    | 使用 `requestAnimationFrame` + 页面不可见时暂停渲染 |
| 模型优化    | 所有 3D 几何体使用低多边形，贴图压缩为 WebP           |
| 着色器复用  | 统一粒子材质，使用 `ShaderMaterial` 减少 draw call  |
| 防抖 resize | Canvas 尺寸变化使用 200ms debounce                    |

### 4.4 Astro 集成注意事项

1. **Three.js SSR 兼容**：所有 `@react-three/fiber` 组件必须添加 `client:only="react"` 或 `client:visible`，避免服务端渲染时报错
2. **分块打包**：配置 Vite `manualChunks`，将 `three` 相关库单独拆分为 chunk，避免首屏加载阻塞
3. **Tailwind 适配**：自然主题通过 CSS 变量注入，Tailwind `@theme` 中注册自然色值
